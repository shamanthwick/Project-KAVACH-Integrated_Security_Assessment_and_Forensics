# Retrospective — Project KAVACH

## ✅ What to KEEP
- Hypothesis-driven analysis approach
- Strong GitHub documentation practices
- Cross-verification between network and web teams

---

## ❌ What to STOP
- Jumping to conclusions without full evidence
- Over-relying on LLM outputs
- Delaying synthesis until final iteration

---

## 🚀 What to START
- Earlier collaboration between workstreams
- Better baseline analysis before anomaly detection
- Continuous validation of assumptions

---

## 🔑 Key Takeaway

The biggest learning was:
> Security analysis is not about finding issues —
> it is about proving them with evidence and connecting them into a coherent story.
